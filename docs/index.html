Inspired by recent work in algorithmic visual style transfer, we set out to build a system that uses reinforcement learning techniques to provide artists with a novel interface for creating music in the style of previous work, reproducing a previous artist‚Äôs ‚Äúsignature‚Äù or ‚Äúessential‚Äù sound. Our intent was to build a system based on promising deep learning approaches to pattern matching for deeply complex but hierarchical data, using such to identify, for a given artist, the major audio frequency components associated with a particular note (and in relation to previous notes), creating an ‚Äúaudiography‚Äù for that artist. Users would then be able to select an artist and create new music in the style of that artist.

While we believe that such a project is possible, and our approach is plausible, the complexity of what we set out to do vastly exceeded the time and computational power we had at our disposal. Nevertheless, our experimentation in this direction yielded a few interesting insights and sounds, which we have loosely organized here.

Approach

Our overall approach was to train a neural net in a ‚Äúreinforcement learning‚Äù style. Briefly, ‚Äúreinforcement learning‚Äù means that you provide the network with an input and an output simultaneously, and train it to produce the output from the given input in the future. A common example of this is image classifier networks: the input is an image of a cat, the output is the label ‚Äúcat‚Äù, and the network is trained to recognize the image, so that in the future you can provide an image of a cat, and the network outputs the label ‚Äúcat‚Äù on its own. By showing enough examples of cats and other objects with the correct labels, the network eventually learns to produce the ‚Äúcat‚Äù label on novel input images.

We are taking the same approach, except that our input is notes played on a keyboard (MIDI) and our output is samples of audio from an MP3. Our intent is for the network to learn what ABBA or Daft Punk ‚Äúsounds like‚Äù for a given note, melody, or chord.

Input

As the input, we used the MIDI standard which normally is used as computer music format in which each individual note that is played is being stored. Midi files then contain instead of the actual audio only the notes that make up a certain song in many ways a very reduced form of the music, a skeleton. Midi is designed to support 127 different notes. The mapping between midi notes and actual frequencies is comparatively simple as each note refers directly to at least one frequency. Output

Signal decomposition into frequencies using a technique called Short-term Fourier Transform (STFT). The STFT technique samples the entire song into really short time chunks (20 milliseconds, 1/50 of a second) and then measures the volume (‚Äúenergy‚Äù) in each sample for each of 1000 frequencies (‚Äútones‚Äù). This yields a ‚Äútable‚Äù where the rows are frequencies of sound, and the columns are each 20-millisecond sample. By using an ‚Äúinverse transform‚Äù we can programmatically reconstruct a perceptually equivalent sound from this table.

These ‚Äútables‚Äù are often displayed as spectrograms, like this example:


What you see above is the relative intensity of the sound in each frequency (Y-axis) over time (X-axis).

However, one thing that the spectrogram doesn‚Äôt capture is that volume is not enough; phase also matters, and it‚Äôs an integral part of the STFT that is hard to learn. Phase is what aligns the various frequencies together so that they sound coherent and produce the actual desired waveform.

Implementation

In this initial approach which can be found in this Notebook we only used linear combinations between the midi input array and the STFT output array.

In this basic example, we used 3 fully connected layers to upscale from the 128 long midi array to the 2050 long array of floats, that contain the real and imaginary parts of the signal decomposition.
